# ddup 产品需求文档 (PRD)

> 版本：1.0.0 | 更新日期：2026-03-11

---

## 一、产品概述

### 1.1 产品定义

**产品名称**：ddup（码姐自我进化系统）

**产品类型**：AI+工具产品

**一句话描述**：为OpenClaw智能体提供自动学习、记忆管理、系统快照、备份还原、安全检查的自动化运维工具。

### 1.2 产品愿景

让AI能够"自我进化"——自主管理知识、记录状态、发现问题、持续学习。

### 1.3 目标用户

| 用户类型 | 需求 |
|----------|------|
| OpenClaw高级用户 | 自动化运维、状态监控 |
| 独立开发者 | AI助手自我管理 |
| 团队管理者 | 多AI协作监控 |

---

## 二、市场分析

### 2.1 痛点来源

通过对OpenClaw小白用户痛点分析，发现以下核心问题：

| 痛点 | 用户占比 | 解决紧迫度 |
|------|----------|------------|
| 记忆缺失 | 90% | 高 |
| 状态不明 | 80% | 高 |
| 备份繁琐 | 70% | 中 |
| 故障难查 | 60% | 中 |

### 2.2 竞品分析

| 竞品 | 优点 | 缺点 |
|------|------|------|
| 手动管理 | 灵活 | 效率低、易出错 |
| 定时脚本 | 自动 | 无统一入口、难维护 |
| ddup | 一体化、零依赖 | 初期功能有限 |

### 2.3 市场机会

- OpenClaw用户量增长
- 自动化需求增加
- 零依赖工具有稀缺性

### 3.3 功能分解

#### F1: 系统快照
```
功能：定期获取OpenClaw状态
输入：-
处理：
  1. 调用 openclaw status
  2. 获取当前时间戳
  3. 写入 snapshots/日期.json
输出：快照JSON文件
```

#### F2: 备份还原
```
功能：备份工作空间配置
输入：-
处理：
  1. 复制核心配置文件
  2. 保存到 backups/日期_时间/
  3. 记录元数据
输出：备份目录
```

#### F3: 安全检查
```
功能：检测核心文件状态
输入：-
处理：
  1. 检查 SOUL.md, AGENTS.md 等
  2. 检测可疑内容
  3. 返回检查结果
输出：检查报告
```

#### F4: Heartbeat
```
功能：一键执行所有增量任务
输入：-
处理：
  1. 执行快照
  2. 执行安全检查
  3. 返回汇总结果
输出：执行报告
```

## 四、产品设计

### 4.1 系统架构

```
┌─────────────────────────────────────────────┐
│              CLI 入口层 (ddup.py)           │
├─────────────────────────────────────────────┤
│              业务逻辑层                       │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │ snapshot │ │  backup  │ │   security   │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
├─────────────────────────────────────────────┤
│              数据存储层                       │
│  snapshots/  backups/  logs/  *.md          │
└─────────────────────────────────────────────┘
```

### 4.2 模块设计

#### 4.2.1 snapshot 模块

| 属性 | 值 |
|------|-----|
| 职责 | 系统快照管理 |
| 依赖 | subprocess, json, datetime |
| 对外接口 | create_snapshot(), get_latest_snapshot() |

#### 4.2.2 backup 模块

| 属性 | 值 |
|------|-----|
| 职责 | 备份还原管理 |
| 依赖 | shutil, json, os |
| 对外接口 | backup_workspace(), restore_backup(), list_backups() |

#### 4.2.3 security 模块

| 属性 | 值 |
|------|-----|
| 职责 | 安全检查 |
| 依赖 | os |
| 对外接口 | security_check() |

### 4.3 数据模型

#### 快照数据
```json
{
  "timestamp": "2026-03-11T09:51:54.051905",
  "date": "2026-03-11",
  "openclaw_status": {
    "returncode": 0,
    "stdout": "...",
    "stderr": ""
  }
}
```

#### 备份元数据
```json
{
  "backup_name": "2026-03-11_095159",
  "date": "2026-03-11",
  "timestamp": "095159",
  "files": ["SOUL.md", "AGENTS.md", "MEMORY.md", "TOOLS.md"]
}
```

### 4.4 接口设计

| 接口 | 输入 | 输出 | 说明 |
|------|------|------|------|
| run | - | JSON | 执行所有任务 |
| snapshot | - | JSON | 创建快照 |
| backup | - | String | 执行备份 |
| security | - | JSON | 安全检查 |
| status | - | JSON | 存储状态 |
| memory | - | List | 记忆列表 |
| list-backups | - | List | 备份列表 |
